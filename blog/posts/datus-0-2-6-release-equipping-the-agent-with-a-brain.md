# Datus 0.2.6 Release: Equipping the Agent with a Brain

<div class="release-026"><div class="page-body"><div style="display:contents" dir="auto"><p id="3299e53b-5536-8091-8838-e1723d3f91a8" class="">If Datus 0.2.5 and earlier versions focused on expanding the underlying toolchain and execution capabilities (such as introducing Skill and MCP mechanisms), the core of 0.2.6 lies in enhancing the Agent&#x27;s autonomous planning and decision-making logic.</p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80d9-b7f3-fe27c5c2dd8e" class="">Building upon the previously established tool ecosystem, 0.2.6 introduces a new General Chat Agent, upgrading its workflow from single, targeted query generation to a complete pipeline of &quot;Intent Understanding -&gt; Task Planning -&gt; Tool Invocation&quot;. Under this architecture, SQL generation is no longer the hardcoded endpoint of the system, but rather one of many tools the Agent can autonomously schedule based on business context.</p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8051-a1e0-f697401b4ddd" class=""><strong>Optimizing the Data Exploration Pipeline and Lowering Contextual Cognitive Barriers</strong></p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80c6-b520-e05abfe7b8f0" class="">In practical data analysis scenarios, the core pain point often lies not in writing SQL syntax, but in sorting out the prerequisite context - such as clarifying unknown table structures, verifying complex metric definitions, or retrieving historical query records. The 0.2.6 release specifically addresses this by refining the Agent&#x27;s upfront data exploration and context confirmation mechanisms, systematically eliminating these prerequisite barriers to analysis.</p></div><div style="display:contents" dir="auto"><h1 id="3299e53b-5536-80f1-af02-f7665145b4ce" class="">Two Core Upgrades</h1></div><div style="display:contents" dir="auto"><ul dir="auto"><ul id="3299e53b-5536-80a9-b206-e56b14f1ac1e" class="bulleted-list"><li style="list-style-type:circle">No longer limited to &quot;only outputting SQL&quot;.</li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-809f-a10b-d3bda641d013" class="bulleted-list"><li style="list-style-type:circle">Upon receiving a task, it autonomously breaks it down and schedules the appropriate tools: whether it&#x27;s inspecting the Schema, searching the knowledge base, reading/writing files, delegating SQL tasks, actively accumulating knowledge, or generating analysis reports, everything is executed in an orderly manner.</li></ul></div></li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-808f-9cee-daf8ae8a2a5b" class="bulleted-list"><li style="list-style-type:disc"><strong>Planning -- Think Before Acting</strong><div style="display:contents" dir="auto"><ul id="3299e53b-5536-8023-be43-d3dba08d8296" style="display:contents" dir="auto"><ul id="3299e53b-5536-80cc-9494-c3bf2d52831a" class="bulleted-list"><li style="list-style-type:circle">Execution proceeds only after manual review and confirmation, providing safety guarantees for cost-sensitive and regulated environments.</li></ul></div></li></ul></div><div style="display:contents" dir="auto"><h1 id="3299e53b-5536-800c-995f-e131fe96399b" Its core lies in endowing the Agent with the ability to proactively retrieve and &quot;observe&quot; the actual content of the database.</p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80ac-90a1-f21cb05ccace" class="">This tool primarily addresses two pain points in actual interaction: first, lowering the cognitive barrier for users when facing unfamiliar databases; second, effectively avoiding blind guessing (hallucinations) by the Agent when lacking underlying data context.</p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8047-b5f0-cd4a9a4a1c49" class="">By introducing this mechanism, the Agent&#x27;s execution pipeline becomes more rigorous and complete. Its workflow shifts to &quot;Preliminary Exploration -&gt; Understand Data Structure -&gt; Generate Query Based on Real Data&quot;. From a technical evolution perspective, this capability is also a further extension and practical application of the semantic tool optimization and knowledge enhancement features from version 0.2.5.</p></div><div style="display:contents" dir="ltr"><figure id="3299e53b-5536-80a0-9ee8-d385a9e0431e" class="image"><a href="/images/datus-0-2-6-release/explore.gif"><img src="/images/datus-0-2-6-release/explore.gif"/></a></figure></div><div style="display:contents" dir="auto"><h2 id="3299e53b-5536-8005-9178-f1491e815243" class="">Plan Mode Optimization -- Long-Range Tasks, Fully Controlled</h2></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8070-a689-cae51cbbdf7b" class="">In version 0.2.5, the system built the initial prototype of Plan Mode by having the Chat node inherit from the GenSQL agentic node. This 0.2.6 release further optimizes Plan Mode. Architecturally, this mode references the design philosophy of Claude Code, and the next version will support Plan persistence.</p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8026-b72b-d6e4dae1263c" class="">This mechanism is primarily used to optimize the execution controllability of complex tasks, and its core workflow includes the following features:</p></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-800d-bbac-d38a70cb3d3f" class="bulleted-list"><li style="list-style-type:disc"><strong>Upfront Planning and Confirmation:</strong> Before calling underlying tools to perform actual operations, the Agent will pre-generate a structured execution plan and submit it to the user to confirm the specific execution method.</li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-802e-98be-ce17f1c9e383" class="bulleted-list"><li style="list-style-type:disc"><strong>Dynamic Progress Visualization:</strong> During task execution, the system dynamically displays the current progress and active node in real-time, significantly reducing the tracking cost of long-range tasks.</li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-809b-81f5-da259b43ac0a" class="bulleted-list"><li style="list-style-type:disc"><strong>Step-by-Step Feedback and Correction:</strong> The execution plan has dynamic adjustment capabilities. At each step of the workflow, the system supports receiving feedback, allowing users to intervene at any time and modify the subsequent plan pipeline.</li></ul></div><div style="display:contents" dir="ltr"><figure id="3299e53b-5536-8050-b944-f018a54974f2" class="image"><a href="/images/datus-0-2-6-release/plan_mode.gif"><img src="/images/datus-0-2-6-release/plan_mode.gif"/></a></figure></div><div style="display:contents" dir="auto"><h2 id="3299e53b-5536-80ed-ae82-e62887b37257" class="">AskUser Tool -- Proactively Ask, Don&#x27;t Guess</h2></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8004-a42f-c7cb42c8a86f" class="">In the existing execution logic of 0.2.5, the Agent tended to directly infer user intent based on implicit assumptions when facing ambiguous inputs; due to the lack of interruption and confirmation mechanisms throughout the execution pipeline, this unidirectional generation mode significantly increased the probability of erroneous queries and analysis results.</p></div><div style="display:contents" dir="ltr"><figure id="3299e53b-5536-8090-893b-e4cabfa7ea2a" class="image"><a href="/images/datus-0-2-6-release/ask_user_0.2.5-1.png"><img src="/images/datus-0-2-6-release/ask_user_0.2.5-1.png"/></a></figure></div><div style="display:contents" dir="ltr"><figure id="3299e53b-5536-8069-83d3-ffc102fdd209" class="image"><a href="/images/datus-0-2-6-release/ask_user_0.2.5-2.png"><img src="/images/datus-0-2-6-release/ask_user_0.2.5-2.png"/></a></figure></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80e8-bc8f-cee8670db6b2" class="">The AskUser Tool introduced in 0.2.6 effectively solves this problem. This tool allows the Large Language Model (LLM) to proactively suspend the current task and initiate a question to the user when it identifies missing context or unclear intent, rather than arbitrarily filling in information. By transforming the workflow from &quot;unidirectional guessing&quot; to &quot;on-demand confirmation&quot;, this mechanism further reduces the risk of AI hallucination from the content generation side. This complements the strengthened parameter validation (String Validation) in 0.2.5, jointly improving the reliability of system outputs.</p></div><div style="display:contents" dir="ltr"><figure id="3299e53b-5536-80e1-968e-dd9edb010283" class="image"><a href="/images/datus-0-2-6-release/ask_user_0.2.6-1.png"><img src="/images/datus-0-2-6-release/ask_user_0.2.6-1.png"/></a></figure></div><div style="display:contents" dir="auto"><h2 id="3299e53b-5536-804d-938e-ca808b31e31b" class="">Storage Pluginization -- Flexible and Extensible</h2></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80d8-a5b6-e361911ac576" class="">With the expansion of Datus, the original storage architecture exposed two structural issues hindering platform development, which directly impacted enterprise-grade self-hosted deployments (such as using PostgreSQL, pgvector, or Milvus) and the construction of the plugin ecosystem:</p></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-806f-be9c-ee2339872465" class="bulleted-list"><li style="list-style-type:disc"><strong>Deep Coupling of Vector Storage (LanceDB Lock-in):</strong> In the original architecture, various modules (such as feedback, metrics, Schema metadata, etc.) directly imported and constructed LanceDB objects. Switching or extending the vector backend meant modifying dozens of scattered files.</li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-80e1-bd47-d02dc2d94697" class="bulleted-list"><li style="list-style-type:disc"><strong>Lack of Abstraction in Relational Storage:</strong> Native SQLite3 calls were scattered across modules like task queues, subject trees, and session storage. Due to the lack of a unified abstraction interface, the system could not smoothly migrate to hosted databases, nor could it establish consistent error-handling boundaries.</li></ul></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-8016-9a3d-f3e02e33b46d" class=""><strong>Core Change: From Hardcoding to Backend-Agnostic Abstraction</strong></p></div><div style="display:contents" dir="auto"><p id="3299e53b-5536-80f9-8ab6-f43caa99c701" class="">This commit completely decouples Datus&#x27;s storage system from any specific database implementation:</p></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-80cc-9a45-f7133786a85f" class="bulleted-list"><li style="list-style-type:disc"><strong>Introduction of a Unified Abstraction Layer:</strong> All internal storage (session history, task queues, reference SQL, etc.) is no longer hardcoded and bound to LanceDB and SQLite, but is entirely placed behind clear, backend-agnostic abstraction interfaces.</li></ul></div><div style="display:contents" dir="auto"><ul id="3299e53b-5536-803b-8482-d1673024b43f" class="bulleted-list"><li style="list-style-type:disc"><strong>Dynamic Pluggability:</strong> Specific underlying database implementations have been transitioned to a pluggable mode.
